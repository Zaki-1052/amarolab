# Input

## Overview[edit](</mediawiki/index.php?title=Input&action=edit&section=1> "Edit section: Overview")]

The input to all of the ensemble builder options is a CSV file. An example is shown below 

  
[![Input csv.png](/mediawiki/images/thumb/0/06/Input_csv.png/800px-Input_csv.png)](</mediawiki/index.php/File:Input_csv.png>)

  
**id** : A unique alphanumeric compound identifier. Each compound should appear only once in the CSV. If a compound has docking scores that correspond to two or more tautomeric or protonation states, then only one should be retained and incorporated into the CSV file. A convenient way to do this is to retain the lowest docking score. 

**status** : The activity of the compound. A "1" denotes an active compound, while a "0" denotes an inactive compound. 

**receptor_1, ..., receptor_N** : Compound docking scores to different conformations of the target. In the current implementation, only docking scores from columns labeled "receptor_#" (where # is a number) will be used during ensemble training. The order in which these columns appear is not important. 

**xtal** : The docking score to the crystal structure. Currently, the xtal column is not included in the training process. If you wish to include the crystal structure to be considered, then it needs to be relabeled; e.g. "receptor_11." 

## Example Input[edit](</mediawiki/index.php?title=Input&action=edit&section=2> "Edit section: Example Input")]

An example input file is available here [input.csv](<https://github.com/rvswift/EB/blob/master/data/andr_merged.csv>). 

The example was generated by clustering 100 nanoseconds of molecular dynamics (MD) simulations of the androgen receptor, [PDBID 2AM9](<http://www.rcsb.org/pdb/explore/explore.do?structureId=2am9>). A total of 10 clusters were produced from the MD simulations. [DUD-E](<http://dude.docking.org/targets/andr>) actives and decoys for the androgen receptor were docked to the cluster centroids and the crystal structure.
