# Exhaustive

### Overview[edit](</mediawiki/index.php?title=Exhaustive&action=edit&section=1> "Edit section: Overview")]

The exhaustive method solves the [ensemble selection problem](</mediawiki/index.php/Training_Ensembles#Overview:_The_Ensemble_Selection_Problem> "Training Ensembles") by brute force enumeration of all possible ensembles. If there are N target conformations, then all of the possible 2N - 1 ensembles are constructed, guaranteeing that the best performing ensemble is identified. Usage information can be generated by typing `ensemblebuilder exhaustive` on the command line, which results in the following output. 
    
    
    <exhaustive> Complete parameter list
    
         --input : (required) csv file of screened molecules.
          Columns should be as follows:
    
              id, status, receptor_1, receptor_2, ..., receptor_N
            CH44,      1,       -9.7,       -9.3, ...,      -10.2
            ZN44,      0,       -6.6,       -6.1, ...,       -6.8
    
           id is a unique molecular identifier
           status takes a value of '1' if the molecule is active and '0' otherwise.
           receptor_1 through receptor_N are docking scores.
    
         --outname : (required) the prefix of the outputfiles.
    
         --ensemble_size : (required) Specifies the maximum ensemble size  to consider.
    
         --fpf : (optional) Rank ensembles by the number of unique active molecules found
           above the specified value. Ensembles will be ranked by AUC if a value  of '1' is
           given. The default is to use the smallest defined false positive fraction.
    
         --ncpu : (optional) Number of parallel processes to use. The default is  to use all
           available cores.

### Command Line Flags[edit](</mediawiki/index.php?title=Exhaustive&action=edit&section=2> "Edit section: Command Line Flags")]

**\--input** : (Required) Example input can be found [here](<https://github.com/rvswift/EB/blob/master/data/andr_merged.csv>), and is described in greater detail [here](</mediawiki/index.php/Input> "Input"). 

**\--outname** : (Required) CSV files that contain the compositions of optimally performing ensembles at discrete sizes are produced. The names of these files have the form "prefix_#_queries.csv". The --outname flag provides the prefix field, and # is given by the ensemble size. For example, if --outname is set to "exh," then the best performing ensembles of size 1 through N will be written to the files exh_1_queries.csv, exh_2_queries.csv, ..., exh_N_queries.csv. The maximum ensemble size is determined by the --ensemble_size flag. 

**\--ensemble_size** : (Required) Optimally performing ensembles up to the value given by --ensemble_size will be determined. For example, if --ensemble_size is set to 5, then optimally performing ensembles of size 1, 2, 3, 4, and 5 will be determined. 

**\--fpf** : (Optional) Ensembles are selected by optimizing a classification performance metric based on the [Receiver Operating Characteristic (ROC) curve](<https://en.wikipedia.org/wiki/Receiver_operating_characteristic>). If the --fpf flag is set to "1," the ensemble that yields the largest [AUC](<https://en.wikipedia.org/wiki/Receiver_operating_characteristic#Area_under_the_curve>), or area under the ROC curve, will be selected at each size. If the --fpf flag is set to a value less than 1, the ensemble that yields the largest ROC enrichment factor at the specified fpf will be selected. The ROC enrichment (EF) is given EF = TPF/FPF. If a value is not specified, the application will use the smallest defined FPF. 

**\--ncpu** : (Optional) Split the calculation across this many processors. If a value is not specified, the application is greedy and will run on all processors. 

### Example[edit](</mediawiki/index.php?title=Exhaustive&action=edit&section=3> "Edit section: Example")]

To generate ensembles of size 1 through 3, optimized to produce the largest AUC, using two cpus, run the following command. 
    
    
    ensemblebuilder exhaustive --input training_set.csv --outname exh --ensemble_size 5 --fpf 1 --ncpu 2

If the command above is run with the example training set found [here](<https://github.com/rvswift/EB/blob/master/data/training_set.csv>), which contains docking scores from 10 receptor conformations, 3 files (exh_1_queries.csv, exh_2_queries.csv, and exh_3_queries.csv) should be produced. They should contain the following contents. 

    **exh_1_queries.csv** : receptor_8
    **exh_2_queries.csv** : receptor_8, receptor_2
    **exh_3_queries.csv** : receptor_10, receptor_8, receptor_2

Each file contains the receptor conformations that produce the largest AUC value of the indicated size. For example, of the 45 possible two-membered ensembles (10 choose 2), receptor_8 and receptor_2 make up the two-membered ensemble that produces the largest AUC value when the training set is screened.
